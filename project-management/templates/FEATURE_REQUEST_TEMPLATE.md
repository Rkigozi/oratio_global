# Oratio Feature Request Template

## Purpose
This template helps you suggest new features for Oratio in a way that our team can understand, evaluate, and potentially implement.

**Before submitting:** Check if your idea aligns with Oratio's mission: "A global Christian prayer platform designed to connect people through shared prayer."

---

## 1. Basic Information

### Contact (Optional)
- **Name:** [Your name]
- **Email:** [Your email - only if you want updates]
- **Role:** [User, Prayer Team Leader, Pastor, Developer, etc.]
- **How you use Oratio:** [Daily, weekly, occasionally, haven't started yet]

### Request Type
- [ ] **New Feature** - Something completely new
- [ ] **Enhancement** - Improvement to existing feature
- [ ] **Integration** - Connect with other tools/services
- [ ] **Accessibility** - Make Oratio more usable for everyone
- [ ] **Performance** - Make Oratio faster/more reliable

### Priority (From Your Perspective)
- [ ] **Critical** - Would significantly increase my use of Oratio
- [ ] **Important** - Would improve my experience a lot
- [ ] **Nice-to-have** - Would be nice but not essential
- [ ] **Future idea** - For long-term consideration

## 2. The Idea

### Feature Name
Give your idea a short, descriptive name:

**Example:** "Prayer Reminders" or "Group Prayer Circles"

### One-Sentence Summary
Describe your idea in one sentence:

**Example:** "Allow users to set reminders to pray for specific requests at later times."

### Detailed Description
Describe your idea in more detail. What would it do? How would it work?

**Example:** "Users could tap a 'Remind me' button on any prayer request and choose when to be reminded (1 hour, 1 day, 1 week). At the chosen time, they'd get a notification prompting them to pray for that request."

---

## 3. The Problem & Solution

### Problem Statement
What problem does this feature solve? Who has this problem?

**Example:** "I often see prayer requests I want to pray for later, but then forget about them. Many Oratio users likely have the same experience of wanting to follow up but losing track."

### Proposed Solution
How does your feature solve this problem?

**Example:** "Reminders would help users follow through on their intention to pray for others, increasing engagement and making prayer support more consistent."

### Alternative Solutions Considered
Are there other ways to solve this problem?

**Example:** "Bookmarks could work, but reminders are more proactive. Email digests could work, but are less immediate."

---

## 4. User Experience

### User Story
Complete this sentence: "As a [type of user], I want to [do something] so that [I get some benefit]."

**Example:** "As an Oratio user, I want to set reminders for prayer requests so that I don't forget to pray for people who need support."

### User Flow
Describe how a user would use this feature:

1. [First step]
2. [Second step]
3. [Third step]
4. [Result]

**Example:**
1. User sees a prayer request they want to pray for later
2. User taps "Remind me" button
3. User selects reminder time (1 hour, 1 day, 1 week)
4. User gets notification at chosen time
5. Notification opens directly to that prayer request

### Visual Mockup (Optional)
If you have design ideas, sketch or describe them:

**Example:** "A small bell icon next to the 'I Prayed' button. Tapping it opens a simple menu with time options."

## 5. Value & Impact

### User Value
How does this help Oratio users?

- [ ] **Saves time** - Makes something faster/easier
- [ ] **Increases engagement** - Encourages more use of Oratio
- [ ] **Deepens connection** - Strengthens prayer relationships
- [ ] **Solves frustration** - Addresses a pain point
- [ ] **Enables new use cases** - Allows something previously impossible

### Business Value
How does this help Oratio as a platform?

- [ ] **Increases retention** - Users return more often
- [ ] **Grows community** - Attracts new users
- [ ] **Differentiates Oratio** - Unique feature vs competitors
- [ ] **Monetization potential** - Could be part of paid features later
- [ ] **Strategic alignment** - Fits Oratio's mission and vision

### Success Metrics
How would we know this feature is successful?

**Example:**
- 20% of users set at least one reminder per week
- Users with reminders pray for 50% more requests
- Positive feedback in user surveys
- Increased daily active users

## 8. Implementation Suggestions

### Phased Approach
Could this be implemented in smaller steps?

**Phase 1:** [Minimal viable version]
**Phase 2:** [Additional features]
**Phase 3:** [Polish and scaling]

**Example:**

**Example:** "Todoist reminders, Google Calendar notifications, prayer reminder apps like Echo."

## 9. Trade-offs & Alternatives

### Simpler Alternative
Is there a simpler version that would still provide most of the value?

**Example:** "Instead of custom reminder times, just have 'Remind me tomorrow' button."

### No-Go Scenario
Under what circumstances should we NOT build this feature?

**Example:** "If notifications cause significant battery drain or privacy concerns."

### What We Might Lose
Does implementing this mean not doing something else?

**Example:** "Building reminders might delay prayer groups feature by 2 weeks."

## 10. For Non‑Technical Submitters

### Don't Worry About
- ❌ Exact technical implementation
- ❌ Database schema details
- ❌ API design
- ❌ Code complexity

### Focus On
- ✅ The user problem you're solving
- ✅ How users would experience the feature
- ✅ Why this matters for Oratio's mission
- ✅ How we'd know it's successful

### Example of a Good Feature Request (Simple)

**Idea:** "Prayer categories filter"
**Problem:** "I want to focus on healing prayers, but have to scroll through everything"
**Solution:** "Add category filters above the prayer feed"
**Value:** "Helps users find prayers relevant to their calling or current needs"

## 11. Evaluation & Next Steps

### How We Evaluate Features
Our team considers:
1. **Alignment with mission** - Does this help people connect through prayer?
2. **User impact** - How many users would benefit? How much?
3. **Feasibility** - Can we build this with our resources?
4. **Priority** - Is this more important than other planned work?
5. **Risks** - Any privacy, safety, or technical risks?

### What Happens Next
1. **Acknowledgement** - We'll confirm we received your request
2. **Discussion** - Team will discuss in weekly planning
3. **Evaluation** - We'll assess against our roadmap
4. **Decision** - We'll decide: Build now, later, or not at all
5. **Update** - We'll let you know the decision (if you provided contact)

### Timeline
- **Immediate:** Requests reviewed within 2 weeks
- **Short-term:** If approved, added to next quarter roadmap
- **Long-term:** May be part of future phases
