# Product Context - Oratio Prayer Platform

## Why This Project Exists
Oratio addresses the human need for spiritual connection and support in an increasingly disconnected world. The platform creates a digital space where people can share their burdens, find community support, and participate in global prayer movements.

## Core Problems Solved
1. **Spiritual Isolation**: People feel alone in their struggles and faith journeys
2. **Prayer Request Fragmentation**: Prayer requests are scattered across social media, messaging apps, and local communities
3. **Global Perspective Lack**: Limited visibility into prayer needs beyond immediate circles
4. **Anonymous Support Need**: Some need prayer but fear judgment or exposure

## Target Users
### Primary Users:
- **Christians seeking prayer support** for personal struggles
- **Prayer warriors** looking to intercede for others
- **Church communities** wanting to extend prayer beyond their walls
- **People in crisis** needing immediate spiritual support

### User Demographics:
- Age: 18-65 (primary: 25-45)
- Geography: Global, English-speaking initially
- Tech Comfort: Moderate to high (smartphone/web users)
- Faith Background: Christian (various denominations)

## How It Should Work (Ideal User Experience)

### Onboarding Flow:
1. **First Visit**: Simple, welcoming interface
2. **Identity Setup**: Choose icon + optional name (no email required)
3. **Immediate Value**: See global prayer activity within 30 seconds

### Core User Journey:
```
Discover → Connect → Participate → Reflect → Return
```

### Key Interactions:
1. **Map Exploration**: Visual discovery of prayer hotspots
2. **Feed Browsing**: Scrollable prayer requests with filtering
3. **Prayer Submission**: Simple form with message + location
4. **"I Prayed" Interaction**: One-tap engagement
5. **Profile Reflection**: View personal prayer journey

## User Experience Goals
### Emotional Goals:
- **Feel connected** to a global community
- **Experience hope** through shared prayer
- **Find comfort** in knowing others are praying
- **Feel empowered** to make a difference

### Functional Goals:
- **Simplicity**: 3 taps or less to complete any action
- **Speed**: Sub-2-second response times
- **Clarity**: Intuitive navigation, no tutorials needed
- **Calm**: Peaceful, non-distracting interface

## Design Principles (From Design-System.md)
### Core Principles:
- **Minimal**: Remove everything non-essential
- **Calm**: Soothing colors, gentle animations
- **Readable**: Clear typography, good contrast
- **Consistent**: Predictable patterns across app

### Visual Language:
- **Colors**: Dark background with soft gradient accents
- **Typography**: Clean sans-serif with clear hierarchy
- **Components**: Rounded, padded, readable
- **Interaction**: Subtle animations, clear feedback

## Safety & Moderation Requirements
### Content Safety:
- **Basic Filtering**: Block obvious abuse/hate/spam
- **Manual Review**: Flagging system for questionable content
- **Safeguarding Disclaimer**: Not a crisis support service
- **Privacy Protection**: No exact locations stored

### Community Guidelines:
- Respectful language only
- No hate speech or discrimination
- No spam or commercial content
- Supportive, encouraging tone

## Success Metrics (User-Focused)
### Engagement Metrics:
- **Daily Active Users (DAU)**: Target: 500+ after 3 months
- **Prayer Submission Rate**: 10% of visitors submit prayers
- **Interaction Rate**: 30%+ click "I Prayed"
- **Return Rate**: 25% return within 7 days

### Quality Metrics:
- **Prayer Response Time**: Average "I Prayed" within 1 hour
- **User Satisfaction**: 4.5+ star rating
- **Content Quality**: Low flag/report rate (<1%)

## Competitive Landscape
### Direct Competitors:
- **Prayer apps** (limited global focus)
- **Social media prayer groups** (fragmented, noisy)
- **Church-specific apps** (limited reach)

### Oratio Differentiators:
1. **Global focus** with map visualization
2. **Anonymous-first** participation
3. **Minimal onboarding** friction
4. **Privacy-focused** design

## Market Opportunity
### Total Addressable Market (TAM):
- **Global Christians**: 2.4+ billion
- **Digital engagement**: 1+ billion smartphone users
- **Prayer app users**: Growing segment

### Serviceable Obtainable Market (SOM):
- **Initial**: English-speaking Christians (400M+)
- **Realistic Year 1**: 10,000+ active users
- **Growth Path**: Language expansion, feature depth

## Business Model Considerations
### MVP Phase (Free):
- No monetization
- Focus on growth and validation
- Optional donations link

### Future Monetization Options:
1. **Church/Organization plans**
2. **Premium features** (advanced analytics, groups)
3. **Donations platform**
4. **Partnerships** (Bible apps, Christian media)

## Risk Factors
### User Risks:
- **Content quality decline** with scale
- **Moderation workload** increase
- **Community management** challenges
- **Privacy concerns** escalation

### Market Risks:
- **Competitor response** from larger platforms
- **User acquisition costs** higher than expected
- **Feature parity** with existing solutions
- **Platform changes** (App Store policies, etc.)

## Vision Evolution
### Phase 1 (MVP):
- Global prayer map + basic feed
- Anonymous participation
- Simple "I Prayed" interactions

### Phase 2 (Community):
- User profiles with stats
- Prayer groups/communities
- Answered prayers tracking

### Phase 3 (Ecosystem):
- Church/organization tools
- Event/prayer campaign features
- API for third-party integration

## Key Assumptions to Validate
1. **Users want global prayer visibility** (vs. local only)
2. **Anonymous participation increases engagement**
3. **Map visualization adds meaningful value**
4. **"I Prayed" interactions provide emotional value**
5. **Users will return without notifications/push**
