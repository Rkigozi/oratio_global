# User Flows

## 1. Purpose

Defines how users move through the application and complete core actions.

---

## 2. Core Loop

enter → explore → submit → pray → reflect → return

---

## 3. Flows

### 3.1 Onboarding
1. open app
2. select icon
3. enter name (optional)
4. begin
5. land on Map

---

### 3.2 Explore (Map → Feed)
1. view map
2. tap marker or navigate
3. open feed

---

### 3.3 Browse Feed
1. scroll prayers
2. filter categories
3. switch tabs

---

### 3.4 Pray
1. tap “I Prayed”
2. count increases
3. confirmation shown

---

### 3.5 Translate
1. tap translate
2. translated text appears

---

### 3.6 Submit Prayer
1. open submit screen
2. write message
3. select category
4. toggle anonymous
5. submit
6. return to feed

---

### 3.7 Profile
1. open profile
2. view stats
3. toggle:
   - My Prayers
   - Prayed For

---

### 3.8 Manage Prayer
1. open own prayer
2. edit / mark answered / delete

---

## 4. Guiding Principle

All flows should feel:

simple → clear → fast → meaningful
